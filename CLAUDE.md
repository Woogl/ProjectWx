# CLAUDE.md

## 기본 지시사항
오픈월드 액션 RPG를 개발하는 클라이언트 프로그래머의 역할을 수행해줘.


## 게임 스펙

- 게임 엔진 : 언리얼 엔진 5.7
- 플랫폼 : PC, 콘솔(PS5)
- 플레이 인원 : 1명(싱글플레이)


## 코딩 규칙

1. 클래스, 구조체, Enum의 Prefix는 `Wx`야. 예) `AWxCharacter`, `FWxPayload`, `EWxCategory`

2. 언리얼 엔진 5의 표준적인 코딩 컨벤션을 준수해서 일관적인 스타일로 작성해.

3. 모든 Gameplay Tag는 C++ 네이티브 태그로 `WxCore` 플러그인에서 관리해.

4. 함수를 선언할 때 인자 개수나 이름이 아무리 길어도 한 줄로 선언해.

5. 딜리게이트에 바인딩하는 콜백 함수 이름은 `Handle`로 시작해. 예) `HandleMontageEnded`, `HandleDeath`

6. Blueprint에서 호출할 이유가 없는 함수에는 `BlueprintCallble` 지정자를 사용하지 마.

7. 헤더에서 inline 함수를 정의하지 마.

8. 람다식은 필수적인 경우, 또는 람다식을 사용하지 않으면 가독성에 문제가 생기는 경우에만 사용해.

9. 코드를 저장소에 제출할 때에는 작업 내용을 한글로 요약 정리해서 올려.


## 신규 개발 플러그인

우리 게임을 구성하는 시스템들을 여러개의 언리얼 엔진 플러그인으로 분리해서 개발할거야.

전체 플러그인 목록은 다음과 같고, `WxCore`를 제외한 플러그인 간에는 직접적인 종속성이 없어야 해.

0. `WxCore` : 공용 태그, 인터페이스 등 공유 정의 전용. 다른 모든 플러그인에서 의존 가능.
1. `WxCombat` : 전투 시스템
2. UI 시스템


### 전투 시스템 (`WxCombat`)

언리얼 GAS 플러그인 기반으로 전투가 가능한 액션 게임 캐릭터를 개발해줘.

타겟팅이 필요하다면 Gameplay Targeting System 플러그인을 사용해.

* 구성 요소
  - 플레이어 캐릭터 클래스
  - 에너미 캐릭터 클래스
  - 어트리뷰트(스탯) 클래스
  - 어빌리티 클래스
  - 무기 클래스
  - 투사체 클래스
