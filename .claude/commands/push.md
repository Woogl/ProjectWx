# 커밋 및 푸시

변경 사항을 커밋하고 원격 저장소에 푸시하라.

## 절차

### 1단계: 변경 사항 확인

다음을 병렬로 실행하라:
- `git status` 로 변경/추가/삭제된 파일 목록 확인
- `git diff` 와 `git diff --staged` 로 변경 내용 확인

변경 사항이 없으면 "커밋할 변경 사항이 없습니다"를 출력하고 종료하라.

### 2단계: 변경 분석

변경된 파일들을 분석하여 다음을 판단하라:
- 어떤 모듈이 변경되었는가 (WxCore, WxCombat, WxUI, WxGame, Docs 등)
- 변경의 성격은 무엇인가 (신규 기능, 리팩터링, 버그 수정 등)

### 3단계: 커밋 메시지 작성

이 프로젝트의 커밋 메시지 컨벤션을 따르라:

```
[모듈명] 변경 내용 요약 (한국어)
```

**규칙**:
- 모듈명은 변경된 모듈을 `/`로 구분: `[WxCombat]`, `[WxCombat/WxCore]`, `[WxGame/WxUI]`
- Content, Config 등 비코드 에셋만 변경된 경우: `[Content]`, `[Config]`
- 문서만 변경된 경우: `[Docs]`
- 요약은 한국어로 작성
- "~를 추가" (신규), "~를 변경/개선" (수정), "~를 수정" (버그), "~를 제거" (삭제), "~를 리팩터링" (구조 변경)

**예시**:
- `[WxCombat] 콤보 시스템을 어빌리티 자체 관리로 리팩터링`
- `[WxCombat/WxCore] 가드 어빌리티 구현 및 HitReact 가드 분기 추가`
- `[Docs] 프로젝트 구조 분석 보고서 추가`

### 4단계: 스테이징 및 커밋

- `.env`, 자격 증명 파일 등 민감한 파일이 포함되어 있지 않은지 확인하라.
- 변경된 파일을 개별적으로 `git add`하라. `git add -A`나 `git add .`는 사용하지 마라.
- HEREDOC 형식으로 커밋하라:

```bash
git commit -m "$(cat <<'EOF'
[모듈명] 커밋 메시지

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
EOF
)"
```

### 5단계: 푸시

- `git push`로 원격 저장소에 푸시하라.
- 현재 브랜치가 원격 추적 브랜치가 없으면 `git push -u origin <브랜치명>`을 사용하라.

### 6단계: 결과 보고

커밋 해시, 메시지, 푸시 결과를 요약하여 보고하라.
